# Cafe Menu Admin

Админ-панель ресторана для официантов.

## Технологии

- **React 18** - UI библиотека
- **TypeScript** - Типизация
- **Vite** - Сборщик
- **React Router** - Маршрутизация
- **Zustand** - Управление состоянием (готово к использованию)
- **Tailwind CSS** - Стилизация
- **LocalStorage** - Хранение данных (готово к замене на API)

## Установка

```bash
npm install
```

## Запуск

```bash
npm run dev
```

## Структура проекта

```
src/
├── components/     # UI компоненты
│   ├── ui/        # Переиспользуемые UI компоненты (Button, Card и т.д.)
│   └── ...        # Специфичные компоненты
├── pages/          # Страницы приложения
├── store/          # Zustand stores (готово к использованию)
├── models/         # TypeScript модели данных
├── services/       # Сервисы для работы с данными
│   ├── api/       # API абстракции (готово к подключению backend)
│   └── *Repository.ts  # Репозитории для работы с данными
├── hooks/          # Переиспользуемые React хуки
├── utils/          # Утилиты и константы
└── styles/         # Глобальные стили
```

## Функциональность

### Основные возможности:
- ✅ Визуальная схема зала со столами и VIP-кабинами
- ✅ Управление меню (блюда и категории)
- ✅ Поиск блюд
- ✅ Бронирование столов и VIP-кабин
- ✅ Редактирование схемы зала (drag & drop)
- ✅ Сохранение данных в LocalStorage

### Оптимизация для официантов:
- ✅ Крупные кнопки и элементы управления (минимум 44px)
- ✅ Оптимизация для touch-устройств (планшеты)
- ✅ Быстрая навигация
- ✅ Четкие цветовые индикаторы статусов
- ✅ Адаптивный дизайн

## Подготовка к подключению Backend

Проект готов к подключению backend API:

1. **API абстракции** - `src/services/api/` содержит интерфейсы для API клиента
2. **Репозитории** - можно легко заменить LocalStorage на API вызовы
3. **Типизация** - все модели данных типизированы

### Для подключения API:

1. Реализовать методы в `src/services/api/client.ts`
2. Обновить репозитории для использования API вместо LocalStorage
3. Добавить обработку ошибок и авторизацию

## Разработка

### Добавление новой страницы:
1. Создать компонент в `src/pages/`
2. Добавить маршрут в `src/router.tsx`
3. При необходимости добавить навигацию в `Header`

### Добавление нового типа данных:
1. Создать модель в `src/models/`
2. Создать репозиторий в `src/services/`
3. Добавить mock-данные в `src/data/mockData.ts`
